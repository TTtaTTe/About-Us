# 📦 보고서 다관점 리뷰 에이전트 v2.0 — 설치 가이드

## 설치 방법 (3단계)

### 1단계: 폴더 찾기

| OS | 경로 |
|----|------|
| Windows | `C:\Users\내사용자이름\.claude\agents\` |
| Mac | `/Users/내사용자이름/.claude/agents/` |

> 💡 `.claude` 폴더가 안 보이면?
> - Windows: 파일 탐색기 → 보기 → "숨긴 항목" 체크
> - Mac: Finder에서 `Cmd + Shift + .` 누르기
>
> 💡 `agents` 폴더가 없으면 직접 만드세요

### 2단계: 파일 복사

`agents/report-reviewer.md` 파일을 위 폴더에 복사합니다.

### 3단계: 확인

Claude Code에서 `/agents` 입력 → `report-reviewer`가 목록에 보이면 완료!

## 사용 방법

### 기본 사용
```
보고서 리뷰해줘
```
리뷰할 파일을 함께 전달하세요.

### 옵션 변경
```
보고서 리뷰해줘 통과점수 9점으로
전략가 관점만 리뷰해줘
100점 만점으로 채점해줘
P1만 보여줘
```

## v2.0 리뷰 프로세스

```
📄 문서 입력
    ↓
📎 맥락 수집 (유형/목적/독자/분야 파악)
    ↓
🔷 3명 리뷰 (전략가 → 분석가 → 독자대변인)
    ↓
📊 리뷰 결과 보고서 제출 ← ★ v2 핵심: 수정 전에 먼저 보고
    ↓
👤 사용자 선택: 수정 적용 / 부분 수정 / 재리뷰 / 논의
    ↓
✏️ 승인된 항목만 반영 → 새 파일 저장
    ↓
📊 전원 8.0+ ? → 완료 / 아니면 다음 라운드
```

## 다른 케이스용 에이전트 만들기

`report-reviewer.md`를 복사한 뒤 리뷰어 페르소나 부분만 수정하면 됩니다:

| 용도 | 리뷰어 변경 예시 |
|------|----------------|
| 기획서 | 전략가→의사결정자, 분석가→실행가능성 검토자, 독자→현업 담당자 |
| 기술문서 | 전략가→시니어 개발자, 분석가→보안 전문가, 독자→주니어 개발자 |
| 마케팅 자료 | 전략가→브랜드매니저, 분석가→데이터 마케터, 독자→타겟 고객 |

## v1 → v2 변경사항

| 항목 | v1 | v2 |
|------|-----|-----|
| YAML frontmatter | 불완전 | name, memory, permissionMode 추가 |
| 모델 | sonnet | opus (1M 컨텍스트) |
| 페르소나 | 한 줄 설명 | 경력/전문성/스타일 2~3문단 |
| 맥락 수집 | 없음 | PHASE 0으로 분리 |
| 수정 프로세스 | 바로 수정 | 보고서 선제출 → 사용자 승인 → 수정 |
| 점수 | 정수 (7/10) | 소수점 허용 (8.7/10.0) |
| 통과 기준 | 7점 | 8.0점 |
| 채점 공식 | 없음 | 투명 공식 공개 |
| 우선순위 | 없음 | P1/P2/P3 체계 |
| 예외 처리 | 없음 | 6가지 시나리오 대응 |
| 메모리 | 없음 | 패턴 축적 (memory: user) |
| 도구 권한 | Read,Write,Edit,Bash,Glob | Read,Write,Glob,Grep (최소 권한) |
